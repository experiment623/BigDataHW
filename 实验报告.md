# 中文欺诈文本检测 — 期末实验报告

> **大数据原理与技术 · 期末项目** | 演讲时长：7 分钟  
> 基于 ChiFraud 数据集的 10 分类欺诈检测 | 字符 N-gram + Transformer 微调 + 跨范式集成

---

## 1. 研究背景与创新点设想（约 1.5 分钟）

### 1.1 任务定义与三大核心挑战

本次实验对中文短信/社交文本进行 **10 分类欺诈检测**：将输入文本判别为 1 类正常 + 9 类欺诈（赌博博彩、招嫖色情、办假证、虚假办卡、违禁药品交易、违规提现、虚假证明、虚假手机卡、地下黑贷）。

**核心挑战**：

1. **语义隐蔽性**：欺诈文本大量使用黑话、形近字、拼音简写（如"办假証"→"办假证"，"vx"→"微信"），传统 jieba 分词容易破坏关键模式。例如 `"办假证"` 可能被切成 `"办 / 假 / 证"` 三个独立词，丢失完整短语信息。

2. **极端类别不平衡**：114,546 条测试集中，正常文本（label=0）占 83.7%，部分欺诈类别仅占 1%~3%（如"办假证"仅 1.4%，"违禁药品"仅 0.8%）。传统模型对少数类的 F1 极低——GAS 在"违禁药品"上 F1 仅 0.5038。

3. **对抗攻击**：8 种攻击策略（形近字替换、随机删字、关键词掩码、拼音简写等）对模型鲁棒性构成严峻考验。对抗集上 Accuracy 从干净测试的 92% 骤降至 50%~69%。

### 1.2 Baseline 的局限性

本项目 5 个 Baseline 均基于传统 NLP 范式：

| Baseline | 核心思路 | 根本局限 |
|:---|:---|:---|
| Word2Vec-w + LR | 词级 Word2Vec + 逻辑回归 | 分词误差传播，OOV 词不可处理，F1-macro 仅 0.41 |
| Word2Vec-c + LR | 字级 Word2Vec + 逻辑回归 | 语义表示能力弱，词表固定，F1-macro 仅 0.47 |
| Word2Vec-c + GBDT | 字嵌入 + 梯度提升树 | 缺乏序列上下文建模，少数类 Recall 极低 |
| Doc2Vec-c + GBDT | 段落向量 + GBDT | 长文本表示退化严重，F1-macro 仅 0.35 |
| GAS (GCN) | 图卷积网络文本分类 | 构图依赖分词，置信度校准极差（F1@90=0.139） |

**关键痛点**：所有 Baseline 都依赖 jieba 分词构建特征，而欺诈文本中的黑话变体在分词阶段就被破坏了。GAS 虽然引入了图结构，但构图仍基于分词，未从根本上解决问题。更严重的是，GAS 的置信度校准极差——在高置信区间 F1@90 仅 0.139，说明模型对其错误预测也高度自信，这在部署中不可接受。

### 1.3 三大创新动机

基于以上分析，我们提出三条创新路线，逐步递进：

**创新一：字符级 N-gram 特征工程 — 绕过分词**

> **核心直觉**：既然分词是瓶颈，为什么不直接绕过分词？  
> 用滑动窗口直接在字符序列上提取 n-gram 特征。`"办假证"` 自然保留为 3-gram，`"vx"` 保留为 2-gram，不被分词破坏。词表从传统 TF-IDF 的 3 万量级跃升至 30~40 万，信息量提升 10 倍以上。配合 `class_weight='balanced'` 缓解类别不平衡。

**创新二：MacBERT/RoBERTa 全词遮罩预训练微调 — 语义理解**

> **核心直觉**：MacBERT 使用全词遮罩（Whole Word Masking）+ 纠错预训练，天然对中文形近字/同音字具备鲁棒性。预训练阶段在大规模语料上学到的语义知识可直接迁移到欺诈检测任务。通过 Focal Loss、WeightedRandomSampler、数据增强等多种策略针对性解决类别不平衡。

**创新三：跨范式多模型加权集成 — 错误互补**

> **核心直觉**：字符 N-gram 模型（8 个变体）擅长捕捉局部字符模式，对黑话变体敏感；Transformer（MacBERT/RoBERTa，8 个变体 × 2 epoch = 14 个投票者）擅长全局语义理解。两种范式的错误模式互不重叠——Transformer 在语义相近但标签不同的文本上可能出错，而字符 N-gram 对局部字符变体更敏感。将 22 个模型统一纳入集成框架，加权融合 + per-class 校正因子 + auto-tune，实现 **1+1>2** 的效果。

## 2. 方法核心机制详解（约 1.5 分钟）

### 2.1 字符级 N-gram SOTA（8 组配置）

**完整流程**：

```
原始文本 → Unicode NFKC 规范化 → 字符级 TF-IDF 向量化 → 分类器预测
                                                                    ↓
                                                           Exact Fallback 覆盖
```

**核心设计决策**：

- **`analyzer='char'`**：不经过分词，直接在字符序列上滑动 n-gram 窗口。这是最关键的创新——绕过了 jieba 分词对欺诈黑话的破坏。

- **`ngram_range = [1, 5]`**：同时保留 1-gram 到 5-gram，覆盖从单字到完整短语的所有粒度。例如 `"办假证"` 同时作为 1-gram（办、假、证）、2-gram（办假、假证）、3-gram（办假证）存在，不同粒度的组合提供冗余和鲁棒性。

- **`sublinear_tf = True`**：对 TF 取 log(1+tf)，抑制高频噪声（如"的"、"了"等常见字）。

- **`max_features = 120k~400k`**：在不同配置中搜索最优特征空间大小，平衡信息量和过拟合风险。

- **分类器多样性**：
  - `LinearSVC`（squared_hinge，C=1/2）：最大间隔分类，适合高维稀疏特征
  - `SGDClassifier`（log_loss，alpha=3e-6）：在线优化，训练快
  - `LogisticRegression`（SAGA solver，C=4）：概率校准好，对抗鲁棒性最强

- **`class_weight='balanced'`**：自动按样本频率反比加权，缓解类别不平衡。

- **Exact Fallback**：训练集中出现过的完全匹配文本直接映射标签，贡献 8,674 条（7.6%）预测。这是一个简单但有效的技巧——欺诈文本经常是模板化的。

**8 组配置对比**：

| 实验 | N-gram | 特征数 | 分类器 | F1-macro |
|:---|:---:|---:|:---|:---:|
| char13_svc_120k | [1,3] | 120k | LinearSVC(C=1) | 0.7383 |
| char14_svc_160k | [1,4] | 160k | LinearSVC(C=1) | 0.7412 |
| char15_svc_c1 | [1,5] | 300k | LinearSVC(C=1) | 0.7508 |
| char15_svc_c2 | [1,5] | 400k | LinearSVC(C=2) | 0.7373 |
| char25_svc_c1 | [2,5] | 300k | LinearSVC(C=1) | 0.7049 |
| hash_char14_sgd_log | [1,4] | 524k(Hash) | SGDClassifier | 0.7002 |
| **char15_sgd_log ★** | [1,5] | 300k | SGDClassifier | **0.7870** |
| char15_lr_saga | [1,5] | 250k | LogisticRegression | 0.6981 |

**关键发现**：
- N-gram 范围 [1,5] 优于 [1,3] 和 [1,4]，更大的上下文窗口捕获了更完整的欺诈短语
- SGDClassifier(log_loss) 在本次任务上优于 LinearSVC，可能因为 log_loss 的概率输出更适合多分类
- char15_lr_saga 的 F1-macro 仅 0.6981，但在对抗集上 Accuracy 达 69.2%（最高），展示了 Logistic Regression 的分布外泛化优势

### 2.2 Transformer 微调（MacBERT / RoBERTa）

**模型选择**：
- `hfl/chinese-macbert-base`：全词遮罩 + 纠错预训练，对形近字/同音字天然鲁棒
- `hfl/chinese-roberta-wwm-ext`：全词遮罩 RoBERTa，更大规模预训练

**训练协议**：train+val → test（利用验证集扩充训练数据），Exact Fallback 启用（覆盖 11,470 条，10.0%）

**8 种不平衡处理方案（每种 2 个 epoch checkpoint = 14 个投票者）**：

| 配置 | 方法 | 核心思想 |
|:---|:---|:---|
| base_train | 仅训练集 | 基线对比 |
| base_+val | 训练+验证 | 利用验证集扩充数据 |
| +val_cwbalanced | class_weight='balanced' | 损失函数中少数类权重反比于频率 |
| +val_cwbalanced_focal1.5 | class_weight + Focal Loss(γ=1.5) | 自动降低易分类样本的损失权重 |
| +val_cwnone_sp0.5 | WeightedRandomSampler(power=0.5) | 过采样少数类，温和缓解不平衡 |
| +val_aug2 | augment-minority=2 | 少数类数据增强，每样本重复 2 次 |

**最优单模型**：`macbert_base_+val_cwnone_sp0.5`（epoch 1），Accuracy 0.9375，F1-macro 0.7594，Recall-macro 0.7481。

**关键发现**：
- WeightedRandomSampler 在 Recall-macro 上表现最好（0.7481），说明过采样策略对提升少数类召回率最有效
- MacBERT 整体优于 RoBERTa，验证了全词遮罩 + 纠错预训练对中文欺诈文本的适配性
- Epoch 2 普遍出现 Recall 下降，说明模型在少数类上开始过拟合

### 2.3 跨范式多模型加权集成（核心创新）

**集成架构**：

```
22 个模型 → 概率向量加权求和 → per-class 校正因子 → argmax → Exact Fallback
  ↑                    ↑                    ↑              ↑
8 字符N-gram     auto-tune搜索        auto-tune搜索    11470条完全匹配
14 Transformer   模型权重[0.03,12]    类别因子[0.03,120]
```

**成员模型（22 个，跨范式融合）**：

- **字符 N-gram 模型（8 个）**：char13/14/15/25_svc、char15_sgd_log、char15_svc_c2、hash_char14_sgd_log、char15_lr_saga
- **Transformer 模型（14 个）**：8 种训练配置 × 1~2 个 epoch checkpoint
  - macbert_base_train (ep1, ep2)
  - macbert_base_+val (ep1, ep2)
  - macbert_base_+val_cwbalanced (ep1, ep2)
  - macbert_base_+val_cwbalanced_focal1.5 (ep1, ep2)
  - macbert_base_+val_cwnone_sp0.5 (ep1, ep2)
  - macbert_base_+val_aug2 (ep1, ep2)
  - roberta_base_+val (ep1, ep2)

**权重搜索策略**：

采用两阶段搜索：
1. **网格搜索**（3 轮）：对每个模型权重和每个类别因子在 [0.03, 12.0] 和 [0.03, 120.0] 范围内乘以 {0.35, 0.50, 0.70, 0.85, 1.0, 1.18, 1.40, 1.75, 2.25} 搜索
2. **随机扰动**（400 次）：对当前最优权重加对数正态噪声，探索网格搜索遗漏的区域

优化目标：`sota_margin` ——在满足所有指标超过已知最高 baseline 的约束下最大化 F1-macro。

**ensemble_cross 搜索到的最优权重**（部分）：
- char13_svc_120k: 2.25（最高权重，字符 N-gram 贡献大）
- char15_sgd_log: 1.652
- macbert_base_+val: 1.0
- roberta_base_+val: 1.0

**Per-Class 校正因子**：
```
[0.70, 11.56, 6.0, 4.0, 2.38, 9.52, 1.03, 1.11, 3.4, 0.51]
```
大幅提升少数类（赌博博彩 ×11.56、违禁药品 ×9.52）的决策权重。

## 4. 消融实验与可视化（约 1 分钟）

### 4.1 集成消融分析（各组件贡献）

```
模型/组件                                    F1-macro    Δ
─────────────────────────────────────────────────────────────
最佳单模型(char15_sgd_log)                      0.7870    baseline
```

## 5. 结论与未来展望（约 1.5 分钟）

### 5.1 核心结论

1. **字符级 N-gram 是最具性价比的创新**：`char15_sgd_log` 在 CPU 上仅需训练，F1-macro 0.787，远超所有 Baseline 最佳（GAS 0.6879，+14.4%）。绕过分词是中文欺诈检测的关键设计选择——直接保留字符序列的 n-gram 模式，避免分词误差传播。

2. **Transformer 微调在 Recall 上有独特优势**：MacBERT + WeightedRandomSampler 达到 Recall-macro 0.7481，比最佳字符模型高 5%。全词遮罩对形近字/同音字的鲁棒性是重要的加分项。但 Transformer 单独使用并未全面超越字符 N-gram——真正质变来自集成。

3. **跨范式集成带来质变**：ensemble_cross 将 8 个字符 N-gram 模型与 14 个 Transformer epoch checkpoint 统一纳入集成框架（共 22 个投票者），加权融合 + per-class 校正因子，将 F1-macro 从最佳单模型 0.787 提升至 **0.868**（+10.3%），在"违禁药品"（+67.8%）、"办假证"（+38.6%）、"赌博博彩"（+35.8%）等少数类上提升尤为显著。跨范式互补是核心驱动力。

4. **置信度校准是部署的关键**：GAS 的 F1@90 仅 0.139（Coverage 70.9%），说明其错误预测被赋予高置信度——这在风控场景中不可接受。ensemble_cross 的 F1@90 达 0.9389（Coverage 90%），F1@95 仍有 0.9297，可以自信地拒绝 5%~10% 不确定样本，同时保持极高准确率。

### 5.2 未来展望

| 方向 | 具体设想 |
|:---|:---|
| **对抗训练** | 在训练阶段注入对抗样本（形近字替换、删字等），提升模型鲁棒性（当前对抗集 Acc 仅 50%~69%） |
| **更大规模预训练模型** | 尝试 ChatGLM/Qwen 等大语言模型的少样本推理能力，利用其更强的语义理解 |
| **数据增强** | 基于黑话词典自动生成同义替换样本，扩充少数类训练数据（当前"违禁药品"仅 0.8%） |
| **多模态融合** | 结合发送者行为特征（频率、时间、社交关系图）进行联合建模 |
| **在线学习** | 部署后持续收集新欺诈模式，通过增量学习保持模型时效性 |
| **知识蒸馏** | 将 22 模型集成知识蒸馏到轻量级单模型中，兼顾性能与推理效率（当前集成推理需加载所有子模型） |
